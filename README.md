# Physics-o1 — Audited Olympiad Corpus and Recipe for Visual Physics Reasoning

NeurIPS 2026 Datasets & Benchmarks Track submission (anonymous).

This repository contains the audit pipeline, dense physics-native reward, training recipe, and evaluation harness used to produce the results reported in the paper.

## Layout

```
audit/                     two-stage 5-gram Jaccard + mxbai-embed-large cosine audit
  audit_two_stage.py         pairwise audit pool against held-out splits
  audit_check.py             diagnostic / per-pair leak inspection
data/
  make_splits.py             build eval_olympiad_v2 + train_pool from raw sources
reward/
  reward_physics.py          5-component dense physics-native reward
                             r = r_ans + r_fmt + r_dim + r_sym + r_cons
                             toggle dense -> binary via env DENSE_REWARD=0
eval/
  eval_batch_phyx.py         PhyX-mini-1k / PhyX-3k MCQ harness (vLLM batched)
  eval_physreason.py         PhysReason-full numerical-multipart harness
  eval_physunibench_oe.py    PhysUniBench-en-OE open-ended harness
  eval_phyx_closed_api.py    PhyX-3k via closed APIs (GPT-4o, Gemini 2.5 Pro)
judge/
  llm_judge_v2_alignment.py  best-match alignment Sonnet judge (PR + PUB-OE)
  llm_judge_v3_pubeo.py      v3 judge with cached gold + CoT-tail fallback (PUB-OE)
  judge_olympiad.py          eval_olympiad_v2 strict + liberal Sonnet judge
LICENSE
README.md
```

## Datasets

- Audited training corpus + 2,434-record clean RL pool:
  [`physics-r1-anonymous/physics-r1-corpus`](https://huggingface.co/datasets/physics-r1-anonymous/physics-r1-corpus)
- **PhysOlym-A** (500-problem open-ended olympiad eval, 99.8% novel-source, two-stage audited):
  [`physics-r1-anonymous/physolym-a`](https://huggingface.co/datasets/physics-r1-anonymous/physolym-a)

Croissant metadata is auto-generated by HuggingFace at the dataset URLs (`/api/datasets/<repo>/croissant`).

```python
from datasets import load_dataset

```

## Reproducing

1. `pip install -r requirements.txt` (vllm==0.11.0, transformers==4.57.0, verl==0.6.1, qwen-vl-utils, datasets, huggingface_hub, mlcroissant).
2. Set environment: `export HF_TOKEN=...` for any HF interactions.
3. Audit your training pool against held-out splits via `audit/audit_two_stage.py`.
4. Train Physics-R1: see Algorithm 3 (recipe) in the paper for the verl launch command — uses `actor_rollout_ref.actor.strategy=fsdp`, `algorithm.adv_estimator=grpo`, `actor_rollout_ref.actor.policy_loss.loss_mode=gspo`, `reward_model.reward_manager=dapo`, `custom_reward_function.path=reward/reward_physics.py`. Toggle dense vs binary via env `DENSE_REWARD=1` (default) or `DENSE_REWARD=0` (binary ablation).
5. Eval: `eval/eval_*.py` produce raw response JSONL; `judge/*.py` Sonnet-grade for strict + liberal scores.
